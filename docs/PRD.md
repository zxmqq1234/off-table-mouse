# 《桌外鼠标》V1.0 产品需求文档 PRD

## 1. 产品名称

桌外鼠标

## 2. 产品定位

将手机浏览器变成电脑的无线触控板、鼠标、键盘和快捷手势控制器。

第一版优先实现局域网内的低延迟鼠标控制、实时键盘输入、快捷键输入、页面滚动、拖拽、边缘持续移动和多指手势操作。

## 3. 第一版目标

用户在电脑端启动程序后，手机扫码打开网页，在同一局域网内连接电脑，并通过手机网页实时控制电脑鼠标、键盘和基础系统快捷操作。

## 4. 第一版技术形态

| 端 | 形态 |
| --- | --- |
| 电脑端 | 桌面客户端 |
| 手机端 | 手机浏览器网页 |
| 通信方式 | WebSocket |
| 连接方式 | 局域网二维码连接 |
| 安全方式 | 一次性 Token + 电脑端确认 |
| 控制方式 | 电脑端模拟鼠标、键盘和快捷键 |

## 6. 核心用户流程

```text
用户打开电脑端程序 → 电脑端启动本地服务 → 生成局域网访问地址和二维码
→ 用户用手机扫码 → 手机浏览器打开控制网页 → 手机端请求连接电脑端
→ 电脑端弹窗确认是否允许连接 → 用户点击允许 → 手机端进入主控制界面
→ 用户通过手机控制电脑鼠标、键盘和快捷操作 → 断开连接或关闭程序
```

## 9. 交互判定规则

### 9.1 鼠标移动
手机端记录 touchmove 的 dx、dy、时间间隔、速度。根据灵敏度和加速度计算最终移动距离。通过 WebSocket 发送 mouse_move 事件。电脑端接收后执行鼠标相对移动。

### 9.2 点击判定
如果手指按下和抬起之间的时间小于点击阈值，且移动距离小于滑动阈值，则判定为点击。

### 9.3 拖拽判定
方式一：单指长按触控区进入拖拽状态，长按后移动手指，手指抬起释放。
方式二：按住顶部左键按钮，另一只手在触控区滑动，松开顶部左键按钮释放。
优先保证方式二稳定可用。

### 9.4 边缘持续移动
当用户手指滑动到触控区边缘并停留超过指定时间，电脑鼠标持续向该方向移动。边缘区域包括上、下、左、右和四个角。手指离开边缘区域或抬起后停止持续移动。

### 9.5-9.7 双指/三指手势
- 双指从左往右滑：Alt + Left（后退）
- 双指从右往左滑：Alt + Right（前进）
- 三指左右滑：Alt + Tab（任务切换）
- 三指从下往上滑：Win + D（回桌面）

### 9.8 实时键盘输入
手机端监听 input 事件。新增字符发送 keyboard_text。删除发送 keyboard_key: Backspace。换行发送 keyboard_key: Enter。中文用 compositionend 等确认上屏后再同步。

## 10. WebSocket 事件协议

### 10.1 通用字段
```json
{
  "type": "event_type",
  "token": "connection_token",
  "clientId": "mobile_client_id",
  "timestamp": 1710000000000,
  "payload": {}
}
```

### 10.2 鼠标移动
```json
{ "type": "mouse_move", "payload": { "dx": 12, "dy": -8, "speed": 1.4 } }
```

### 10.3 鼠标点击
```json
{ "type": "mouse_click", "payload": { "button": "left", "clicks": 1 } }
```

### 10.4 鼠标按下
```json
{ "type": "mouse_down", "payload": { "button": "left" } }
```

### 10.5 鼠标松开
```json
{ "type": "mouse_up", "payload": { "button": "left" } }
```

### 10.6 鼠标滚动
```json
{ "type": "mouse_scroll", "payload": { "deltaX": 0, "deltaY": -120 } }
```

### 10.7 实时文本输入
```json
{ "type": "keyboard_text", "payload": { "text": "你" } }
```

### 10.8 单个按键
```json
{ "type": "keyboard_key", "payload": { "key": "Enter" } }
```

### 10.9 快捷键
```json
{ "type": "shortcut", "payload": { "keys": ["Ctrl", "C"] } }
```

### 10.10 手势事件
```json
{ "type": "gesture", "payload": { "gesture": "two_finger_swipe_right" } }
```

### 10.11 边缘持续移动
```json
{ "type": "edge_move_start", "payload": { "direction": "right", "speed": 1.0 } }
{ "type": "edge_move_stop", "payload": {} }
```

### 10.12 心跳
```json
{ "type": "ping", "payload": {} }
{ "type": "pong", "payload": {} }
```

### 10.13 断开连接
```json
{ "type": "disconnect", "payload": { "reason": "user_disconnect" } }
```

## 12. 非功能需求

### 12.1 性能
鼠标移动延迟应尽量低。WebSocket 消息应轻量。高频 mouse_move 事件需要节流或合并。手机端触控反馈需要流畅。电脑端不能因高频输入事件卡顿。

### 12.2 兼容
第一版优先支持 Windows。手机网页优先兼容 Chrome、Safari、Edge。手机端优先适配 iOS Safari 和 Android Chrome。页面需要适配不同手机屏幕尺寸。

### 12.3 容错
端口被占用时提示用户。获取不到局域网 IP 时提示用户检查网络。手机和电脑不在同一局域网时提示无法连接。权限不足时提示用户授权。WebSocket 断开时提示连接已断开。鼠标或键盘模拟失败时提示异常。

### 12.4 安全
默认只允许局域网访问。不默认开放公网访问。二维码 Token 必须有过期时间。手机首次连接必须经过电脑端确认。第一版只允许单手机连接。用户可以随时断开连接。
